# Подготовка к экзамену — Разработка WEB-приложений

Материалы оформлены как учебный GitHub-репозиторий по экзаменационным билетам.

## Как пользоваться

1. Открой папку `tickets`.
2. Начинай с `ticket-01.md`.
3. На каждый билет учи:
   - простое объяснение;
   - готовый ответ на 1 теоретический вопрос;
   - готовый ответ на 2 теоретический вопрос;
   - практическое задание;
   - краткую версию для устного пересказа;
   - типичные ошибки;
   - мини-тренировку.

## Рекомендуемый план на 4 дня

### День 1

## Главная архитектурная цепочка

```text
Web client
→ gRPC wrapper
→ backend gRPC API
→ service / Logic
→ Repository
→ PostgreSQL / Redis / Fias API
```

## Главная цепочка pub/sub

```text
Издатель
→ событие
→ сервис подписок
→ Kafka / очередь
→ Подписчик 1 / Подписчик 2 / Подписчик 3
```

## Что нужно говорить на экзамене чаще всего

```text
Клиент не работает напрямую с БД.
Клиент обращается к backend через gRPC.
Backend выполняет валидацию и бизнес-логику.
Repository скрывает конкретную базу данных.
DTO используется для передачи безопасных данных.
Log нужен для диагностики.
Audit нужен для истории важных действий.
Kafka — поток событий.
RabbitMQ — очередь задач.
Redis — быстрый кэш.
```
