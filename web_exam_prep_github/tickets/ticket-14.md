# Билет №14 — Fias API, города, нормализация адреса

## 0. Как понять этот билет простыми словами

Fias API нужен, чтобы адреса были не произвольной строкой, а нормализованными данными.

```text
"Москва, ул. Ленина, 1" → city_id + street + house
```

---

## 1. Теоретический вопрос 1

**Вопрос:** Что такое Fias API? Для каких задач используется интеграция с ФИАС в CRM-приложении?

### Простое объяснение

ФИАС — справочник адресов. Fias API позволяет искать города, улицы, дома, проверять адреса и получать официальные идентификаторы.

### Готовый ответ на 1 вопрос

Fias API — это внешний интерфейс для работы со справочником адресов ФИАС. Он позволяет искать города, улицы и дома, проверять корректность адреса и получать официальные идентификаторы адресных объектов.

В CRM интеграция с Fias API используется для нормализации адресов пользователей, выбора города из справочника, уменьшения ошибок ручного ввода и хранения в базе структурированных данных, например `city_id`, улицы и дома.

---

## 2. Теоретический вопрос 2

**Вопрос:** Как организован справочник «Города» в приложении? Как связаны локальная таблица городов и внешний Fias API?

### Простое объяснение

В приложении может быть таблица `cities`: `id`, `fias_id`, `name`, `region`, `updated_at`.

Сначала backend ищет город локально или в Redis. Если нет — обращается к Fias API, сохраняет результат и использует дальше.

### Готовый ответ на 2 вопрос

Справочник «Города» в приложении может быть организован как локальная таблица, связанная с внешним Fias API. В локальной таблице хранятся внутренний id, `fias_id`, название, регион и дата обновления.

Когда пользователь вводит город, backend сначала проверяет локальную таблицу или Redis-кэш. Если нужного города нет, система обращается к Fias API, получает официальные данные, сохраняет их локально и использует в дальнейшей работе.

## 3. Практическое задание

**Задание:** Напишите алгоритм нормализации адреса пользователя.

### Решение

```text
1. Пользователь вводит: "Москва, ул. Ленина, 1".
2. Frontend отправляет адрес на backend.
3. Backend разбирает строку: город, улица, дом.
4. Backend ищет город в cities/Redis.
5. Если города нет — запрашивает Fias API.
6. Fias API возвращает официальный city/fias_id.
7. Backend сохраняет город локально.
8. Backend сохраняет адрес: user_id, city_id, street, house.
9. Backend возвращает нормализованный адрес.
```

```ts
async function normalizeAddress(userId: string, rawAddress: string): Promise<void> {
    const parsed = parseAddress(rawAddress);

    let city = await cityRepository.findByName(parsed.city);

    if (!city) {
        const fiasCity = await fiasApi.findCity(parsed.city);

        city = await cityRepository.save({
            fiasId: fiasCity.id,
            name: fiasCity.name,
            region: fiasCity.region
        });
    }

    await addressRepository.save({
        userId,
        cityId: city.id,
        street: parsed.street,
        house: parsed.house
    });
}
```

## 4. Краткая версия для устного пересказа

Fias API — внешний сервис для официального справочника адресов. В CRM он нужен для проверки и нормализации адресов. Система хранит не только строку, а структурированные данные: city_id, street, house. Локальная таблица городов и Redis уменьшают количество обращений к Fias API.

---

## 5. Что обязательно сказать

- Fias API нужен для адресов и городов.
- Нормализация = привести адрес к единому виду.
- Лучше хранить city_id.
- Можно использовать cities и Redis.
- Backend обращается к Fias API.

---

## 6. Типичные ошибки

1. Хранить весь адрес одной строкой.
2. Не сохранять city_id.
3. Запрашивать Fias API прямо с frontend.
4. Не использовать кэш/локальную таблицу.
5. Забыть случай, когда адрес не найден.
