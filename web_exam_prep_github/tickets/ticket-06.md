# Билет №6 — UI/UX, accessibility, wireframe личного кабинета

## 0. Как понять этот билет простыми словами

UI — как интерфейс выглядит. UX — насколько удобно пользователю выполнить задачу.

Для подписчика главное: видеть активные подписки, кнопку отмены и уведомления.

---

## 1. Теоретический вопрос 1

**Вопрос:** Перечислите основные принципы UI/UX для WEB-приложений. Как они применяются при проектировании интерфейса подписчика?

### Простое объяснение

Принципы UI/UX: простота, понятная навигация, единый стиль, обратная связь, минимум лишнего, понятные ошибки, адаптивность.

В кабинете подписчика список подписок должен быть виден сразу, кнопка отмены — рядом с подпиской, после действий должно быть уведомление.

### Готовый ответ на 1 вопрос

Основные принципы UI/UX для WEB-приложений — это простота интерфейса, понятная навигация, единый визуальный стиль, обратная связь после действий пользователя, адаптивность и понятная обработка ошибок.

В интерфейсе подписчика эти принципы применяются так: список активных подписок должен быть виден сразу, кнопка отмены должна находиться рядом с каждой подпиской, статусы должны быть понятными, а после действий система должна показывать уведомление об успехе или ошибке.

---

## 2. Теоретический вопрос 2

**Вопрос:** Что такое доступность WEB-форм? Какие атрибуты HTML и практики следует использовать для формы подписки?

### Простое объяснение

Accessibility — доступность интерфейса для разных пользователей, включая тех, кто использует клавиатуру или скринридер.

Нужно использовать `label`, правильные `type`, `required`, понятные ошибки, фокус клавиатуры, aria-атрибуты при необходимости.

### Готовый ответ на 2 вопрос

Доступность WEB-форм означает, что формой могут пользоваться разные пользователи, включая людей со слабым зрением или тех, кто использует клавиатуру и скринридер. Для этого нужно использовать семантический HTML и корректные атрибуты.

В форме подписки каждому полю нужно задать `label`, использовать правильные типы `input`, например `type="email"`, добавлять `required`, связывать ошибки с полями через `aria-describedby`, обеспечивать фокус клавиатуры и показывать ошибки понятным текстом.

## 4. Краткая версия для устного пересказа

UI/UX отвечает за удобство и понятность. В личном кабинете нужно сразу показать активные подписки, статусы, кнопку отмены и уведомления. Accessibility — доступность формы для всех пользователей: нужны label, правильные типы полей, required, понятные ошибки и управление с клавиатуры.

---

## 5. Что обязательно сказать

- UI = внешний интерфейс.
- UX = пользовательский опыт.
- Accessibility = доступность.
- label обязателен.
- Кнопка отмены рядом с подпиской.

---

## 6. Типичные ошибки

1. Не разделять UI и UX.
2. Забыть блок уведомлений.
3. Спрятать кнопку отмены.
4. Не использовать label.
5. Писать ошибки техническим языком.
